OpenClaw injects a fixed set of .md files from the workspace root into every model run as "Project Context." Our workspace root had 23 markdown files — 230,000 chars total — against a 60,000-char budget. DIRECTIVES.md alone was 42,561 bytes, consuming 70% of the budget every single turn.

**Verified from openclaw.json:**
- `bootstrapMaxChars: 16000` (per file)
- `bootstrapTotalMaxChars: 60000` (total)

**Permanent rule needed in AGENTS.md (Joseph approval pending):**
Only these files belong in workspace root: AGENTS.md, SOUL.md, TOOLS.md, IDENTITY.md, USER.md, HEARTBEAT.md, MEMORY.md, STATUS.md. Everything else goes in `reference/` or subdirectories. Files in subdirectories are NOT auto-injected; must be loaded explicitly with `cat`.

**Why it keeps happening:** No enforcement rule existed. Files get created in root during active build sessions because it's convenient. Needs an explicit rule + self-monitoring in STATUS.md.
