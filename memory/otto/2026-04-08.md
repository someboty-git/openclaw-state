## Critical distinction confirmed: openclaw cron edit vs openclaw.json

- `openclaw cron edit` = safe. Goes through gateway API. Stores in `~/.openclaw/cron/`. Gateway keeps running if it fails.
- `openclaw.json` direct edits = deadlock zone. Controls gateway startup. Wrong change = gateway can't start = Otto offline.
- The April 1 crashes were caused by `gateway.bind` changes in openclaw.json — NOT by cron edits.
- Cron model overrides are NOT in openclaw.json. They're in separate cron job records.
- Rule confirmed: `openclaw cron edit` is always safe. `openclaw.json` is always deadlock zone. These are different things.
